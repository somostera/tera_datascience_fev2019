Como são formadas as APIs REST¶

Podemos dizer que uma API como se fosse uma pgina da web s que para computadores. S eles conseguirem acessar, ler e interagir da maneira adequada com as APIs.

Como Data Scientists, vocs utilizaro APIs tanto internas (fornecidas pelas prprias empresas em que vocs trabalham) quanto externas.

APIs so uma das principais formas que um Data Scientist utiliza para obter os dados que devem ser analisados.

Como exemplo, ao acessarmos a API da Jsonplaceholder https://jsonplaceholder.typicode.com/posts/1, vemos o retorno abaixo, ao invs de uma pgina comum:

{
  "userId": 1,
  "id": 1,
  "title": "sunt aut facere repellat provident occaecati excepturi optio reprehenderit",
  "body": "quia et suscipit\nsuscipit recusandae consequuntur expedita et cum\nreprehenderit molestiae ut ut quas totam\nnostrum rerum est autem sunt rem eveniet architecto"
}

Esse padro de resposta o que chamamos de JSON (JavaScript Object Notation - Notao de Objetos JavaScript), e este o tipo de resposta mais comum entre as APIs.

Ele foi feito para que mquinas entendam seus dados e os utilizem como quiser.

Como funciona o protocolo HTTP¶

A internet trafega atravs do protocolo HTTP (Hypertext Transfer Protocol). O protocolo HTTP foi criado por Tim Berners-Lee, o criador da web, enquanto ele trabalhava na CERN em 1989 (sua equipe e ele tambm foram responsveis pelo HTML).

O HTTP possui duas entidades bsicas: a Request (pedido) e a Response (resposta)

Sempre que entramos em uma pgina, enviamos uma request para o servidor do site e esperamos uma response com o HTML da pgina.

Sempre que enviamos um Request (pedido) devemos especificar um mtodo. A tabela abaixo cita os mtodos mais comuns e explica brevemente o objetivo de cada um:

Mtodo Objetivo
GET Obter informaes
POST Salvar informaes
PUT Atualizar informaes
DELETE Deletar informaes

Todos eles so comumente utilizados por APIs.

O mtodo GET o que seu navegador usa para baixar o HTML da pgina.

O metdo POST tambm muito utilizado sempre que voc envia algum formulrio em um site.

Alm do mtodo, as requisies tambm possuem um header:

O Header (cabealho) tem a funo de dizer para o servidor diversos dados sobre quem fez a solicitao. Dessa forma, o servidor consegue at permitir que voc veja mais informaes dependendo do seu nvel de acesso.

Todos as respostas possuem, alm do contedo da resposta, um cdigo de status.

Os cdigos de status (status code) seguem um padro.

Os mais comuns so:

Cdigo Significado
200 Ok
3XX O servidor vai te redirecionar para outra URL
4XX O recurso no foi encontrado, no est mais disponvel, etc
5XX Algum problema aconteceu no servidor e ele no conseguiu processar a sua request

Outra caracterstica das APIs REST a diviso dos recursos. Por exemplo:

https://jsonplaceholder.typicode.com/users/1

Ao abrirmos esta URL estamos fazendo um GET para obtermos informaes do usurio (/users) com cdigo 1 (/1). Podemos trocar o cdigo do usurio para buscar outros usurios.

Tambm podemos pedir todos os usurios digitando:

https://jsonplaceholder.typicode.com/users

Tambm muito comum o uso de parmetros na URL para passarmos filtros para a API. Por exemplo:

https://jsonplaceholder.typicode.com/comments?postId=1

Perceba o "?postId=1" ao final da URL.

Sempre que precisamos passar parmetros, comeamos com um ponto de interrogao (?) e depois utilizamos o & para parmetros futuros. Por exemplo:

https://jsonplaceholder.typicode.com/comments?postId=1&id=3

Escrita e leitura de arquivos em Python¶

Em Python, assim como em outras linguagens de programao, podemos manipular arquivos que esto em nosso disco.

Para isso, utilizamos o open para nos trazer um objeto de arquivo.

O open precisa saber o nome do arquivo para conseguir ler seu contedo:

open("nome_do_arquivo")

No entanto, existe um segundo argumento que podemos passar para dizer qual o modo que queremos trabalhar com esse arquivo.

Abaixo podemos ver exemplos de cdigos com os modos mais comuns:

open('nome_do_arquivo', 'r')  #Apenas leitura - padro quando o modo  omitido

open('nome_do_arquivo', 'r+') #Leitura e escrita

open('nome_do_arquivo', 'w')  #Apenas escrita, apaga tudo que tem no arquivo antes de escrever novamente

open('nome_do_arquivo', 'a')  #"Append", adiciona o que voc quiser ao final do arquivo, sem apagar nada

Tambm existe um argumento para passarmos em qual encoding gostaramos de escrever.

open('nome_do_arquivo', 'w', encoding='utf-8')

Voc pode conferir todos os parmetros do mtodo open aqui: https://docs.python.org/3/library/functions.html#open

Todo o processo de escrita e leitura de arquivos deve ser feita com muito cuidado. Como so operaes mais lentas, voc pode acabar travando seu computador se fizer um loop errado enquanto escreve um arquivo.

Por isso, muito importante se lembrar de SEMPRE fechar o arquivo aps o uso dele no Python.

Vamos ver um exemplo:

f = open('arquivo_teste.txt', 'w')

f.write('Al? Algum me escuta?')

f.close()

Porque o Context Manager (with) é necessário¶

Em Python, podemos pedir ajuda de um Context Manager para automaticamente fechar o arquivo aps o uso.

Isso a recomendao das boas prticas e evita que seu cdigo tenha bugs.

Para utilizarmos essa funcionalidade, basta utilizar a palavra reservada with

Vamos ver um exemplo:

with open(('arquivo_teste.txt''arquivo , 'r') as f:
    print(f.readline())

print('\n\nSer que o arquivo est fechado? {0}'.format(f.closed))

O que é um arquivo CSV (Comma-separated values)¶

Arquivos CSV so apenas uma coleo de dados separados por um caractere especial (normalmente , ou ;).

Eles so muito importantes para fazer movimentaes de dados de um lugar para o outro.

Todos os softwares de planilha do mercado aceitam arquivos CSV.

Alguns cuidados que devemos seguir ao criar um arquivo CSV:

  • Sempre terminar as linhas com \n
  • Sempre utilizar o encoding como utf-8
  • Utilizar caracteres que no influenciam no dado (ex.: vrgulas para separar dados nmericos que tambm usam vrgulas)

Vamos ver um exemplo de como podemos criar um arquivo CSV em Python a partir do scraping da API do Jsonplaceholder.
